## ⚡ 빠른 진행을 위한 팁

### 🔥 병목 지점 (시간 오래 걸림)
1. **Step 03 (1-2일)**: 공통 엔진 구현 - 가장 복잡
2. **Step 08 (1-2일)**: 알림 시스템 - 테스트 어려움
3. **Step 13 (2-3일)**: 배포 - 빌드 및 스토어 제출

### ⏩ 시간 단축 방법
1. **Step 03 완료 후**: Step 04~06 병렬 진행 (팀 개발)
2. **Firebase Emulator 사용**: Step 10 테스트 시간 단축
3. **EAS Build 미리 설정**: Step 13 진입 전 계정 준비

### ⚠️ 주의해야 할 지점
1. **Step 03 → 04 전환**: 공통 엔진 테스트 완벽히 통과 확인
2. **Step 06 → 07 전환**: 모든 모듈 CRUD 작동 확인
3. **Step 09 → 10 전환**: 전체 앱 플로우 테스트 완료
4. **Step 12 → 13 전환**: 개인정보처리방침 URL 배포 완료

## 📌 체크포인트

### ✅ Phase 1 완료 체크 (Step 03 후)
```bash
# 1. 앱 실행
npx expo start
# 2. 테스트 통과
npm test
# 3. Firestore 연결
# Firebase Console에서 users 컬렉션 확인
```

### ✅ Phase 2 완료 체크 (Step 06 후)
```bash
# 각 모듈 화면에서
# 1. 청소 테스크 생성 → 완료 → 다음 due 계산
# 2. 식재료 추가 → 유통기한 자동 계산
# 3. 약 추가 → 복용 알림 스케줄링
```

### ✅ Phase 3 완료 체크 (Step 09 후)
```bash
# 전체 플로우 테스트
# 1. 온보딩 → 홈 화면 (5분 내)
# 2. 홈 → 청소 → 냉장고 → 약 (모든 화면 이동)
# 3. 테스크 완료 → 점수 반영 확인
# 4. 60fps 유지 (React DevTools Profiler)
```

### ✅ Phase 4 완료 체크 (Step 11 후)
```bash
# 1. Security Rules 배포
firebase deploy --only firestore:rules
# 2. 개인정보처리방침 URL 접근
# 브라우저에서 URL 열기
# 3. 계정 삭제 기능
# 설정 → 계정 삭제 → Firebase에서 삭제 확인
```

### ✅ Phase 5 제출 전 체크 (Step 13)
```bash
# 1. 전체 테스트 통과
npm test
# 2. 30분 이상 실제 기기 사용
# 크래시 없음 확인
# 3. 스토어 메타데이터 준비
# 스크린샷, 설명, 키워드
# 4. 빌드 성공
eas build --profile production --platform all
```
