# 다정한 앱 - 로컬 실행 가이드

## 🚀 빠른 시작

### 1. 앱 실행

```bash
cd /Users/edwardshin/Desktop/dev/dajeonghan/dajeonghan-app

# Expo 개발 서버 시작
npm start
# 또는
npx expo start
```

### 2. 실행 옵션

터미널에서 다음 키를 눌러 플랫폼을 선택하세요:

- **`i`** - iOS 시뮬레이터에서 실행
- **`a`** - Android 에뮬레이터에서 실행
- **`w`** - 웹 브라우저에서 실행 (제한적 기능)

또는 Expo Go 앱으로 QR 코드를 스캔하세요.

## 📱 플랫폼별 설정

### iOS (macOS 전용)

```bash
# iOS 시뮬레이터 직접 실행
npm run ios
# 또는
npx expo start --ios
```

**요구사항**:
- macOS
- Xcode 설치됨
- iOS Simulator 설정됨

### Android

```bash
# Android 에뮬레이터 직접 실행
npm run android
# 또는
npx expo start --android
```

**요구사항**:
- Android Studio 설치됨
- Android SDK 설정됨
- 에뮬레이터 생성됨

### 실제 기기에서 테스트

1. **Expo Go 앱 설치**
   - iOS: App Store에서 "Expo Go" 다운로드
   - Android: Play Store에서 "Expo Go" 다운로드

2. **QR 코드 스캔**
   - `npm start` 실행 후 표시되는 QR 코드 스캔
   - 같은 Wi-Fi 네트워크에 연결되어 있어야 함

## 🧪 테스트 시나리오

### 1. 온보딩 플로우 테스트

1. 앱 첫 실행
2. "시작하기" 버튼 클릭
3. Kakao 로그인 시도 (실제 로그인 또는 스킵)
4. 페르소나 선택 (예: "바쁜 직장인")
5. 온보딩 완료 확인

**예상 결과**: 홈 화면으로 이동

---

### 2. 냉장고 모듈 테스트

1. 홈 화면에서 "냉장고" 탭 선택
2. "+" 버튼으로 식재료 추가
   - 이름: 우유
   - 카테고리: 유제품
   - 유통기한: 오늘 + 7일
3. 저장 버튼 클릭
4. 목록에서 추가된 우유 확인
5. 우유 클릭하여 상세 화면 확인
6. 수정 또는 삭제 테스트

**예상 결과**: 
- Firebase에 데이터 저장됨
- 목록에 즉시 반영됨
- 유통기한 임박 시 알림 표시

---

### 3. 청소 모듈 테스트

1. 홈 화면에서 "청소" 탭 선택
2. 기본 테스크 확인 (온보딩 시 생성됨)
3. 테스크 하나 선택
4. "완료" 버튼 클릭
5. 완료 애니메이션 확인
6. 통계 업데이트 확인

**예상 결과**:
- 테스크 상태 변경
- completionHistory에 기록
- 다음 예정일 자동 계산

---

### 4. 약 관리 테스트

1. 홈 화면에서 "약" 탭 선택
2. "+" 버튼으로 약 추가
   - 약 이름: 비타민 C
   - 복용 시간: 아침 9시
   - 주기: 매일
3. 저장 후 목록 확인
4. 복용 체크박스 클릭
5. 복용 기록 확인

**예상 결과**:
- 약 정보 저장
- 복용 시간에 알림 예약
- 복용 기록이 doseLogs에 저장

---

### 5. 설정 및 프로필 테스트

1. 홈 화면에서 "설정" 탭 선택
2. 프로필 정보 확인
3. 알림 설정 변경
4. "개인정보처리방침" 링크 클릭
5. 웹뷰에서 정책 문서 확인
6. "로그아웃" 버튼 테스트 (선택)

**예상 결과**:
- 설정 변경사항 저장
- 외부 링크 정상 작동
- 로그아웃 시 온보딩 화면으로

---

### 6. 오프라인 모드 테스트

1. 앱 실행 중 비행기 모드 켜기
2. 냉장고에서 식재료 추가 시도
3. 청소 테스크 완료 시도
4. 비행기 모드 끄기
5. 자동 동기화 확인

**예상 결과**:
- 오프라인에서도 기본 기능 작동
- 네트워크 복구 시 자동 동기화
- 데이터 손실 없음

---

### 7. 알림 테스트

1. 설정에서 알림 권한 확인
2. 청소 테스크의 알림 설정
3. 예정 시간 1-2분 후로 설정
4. 앱을 백그라운드로 이동
5. 알림 수신 대기

**예상 결과**:
- 예정 시간에 푸시 알림 수신
- 알림 클릭 시 앱 열림
- 해당 테스크로 이동

### 8. 습관 추적 테스트

1. 3일 연속 테스크 완료

## 🆘 문제 해결

### Q1: 앱이 실행되지 않음

```bash
# 캐시 삭제 후 재시작
npx expo start --clear
```

### Q2: Firebase 연결 오류

1. `.env` 파일 확인
2. Firebase 프로젝트 ID 확인
3. 네트워크 연결 확인

### Q3: 빌드 오류

```bash
# node_modules 재설치
rm -rf node_modules
npm install

# Expo 캐시 삭제
npx expo start --clear
```

### Q4: iOS/Android 실행 오류

```bash
# iOS
cd ios && pod install && cd ..

# Android
cd android && ./gradlew clean && cd ..
```
